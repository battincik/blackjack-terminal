# Contributing Guide

We thank everyone who is interested in contributing to our project. The following rules are designed to make the contribution process efficient and orderly for everyone.

## Contribution Steps

1. Fork the project and clone it to your computer.
2. Create a new branch for adding a new feature or fixing a bug.
3. Make your changes and ensure that the code works.
4. Commit your changes and push them to your fork.
5. Submit a pull request to the main project.

## Coding Standards

- Strive to write clean, readable, and consistent code.
- Add comments and documentation as necessary for new features.
- Write and ensure existing tests pass.

## Issues and Requests

If you find a problem or have a feature request, please first check the existing issues and requests. If there is no similar issue or request, open a new one.

Thank you!
